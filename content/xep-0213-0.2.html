1. Introduction

The XMPP Standards Foundation (XSF) [1] defines protocol suites for the purpose of compliance testing and software certification. This document specifies the XMPP Intermediate Client 2008 certification level.

2. Definition

The XMPP Intermediate Client 2008 certification level is defined as follows:

Table 1: Required and Recommended Specifications

Specification Requirement Level
XMPP Basic Client 2008 [2] REQUIRED
Server-Based Privacy Rules [3] RECOMMENDED
Multi-User Chat [4], Entity Use Cases and Occupant Use Cases REQUIRED
vcard-temp [5] RECOMMENDED
XHTML-IM [6] REQUIRED
Chat State Notifications [7] REQUIRED

Note: These protocols have their own dependencies, which include the following XEPs (as well as various IETF RFCs and W3C specifications):

  • Data Forms [8]
  • Field Standardization for Data Forms [9]
  • XMPP Date and Time Profiles [10]

3. Security Considerations

This document introduces no additional security considerations above and beyond those defined in the documents on which it depends.

4. IANA Considerations

This document requires no interaction with the Internet Assigned Numbers Authority (IANA) [11].

5. XMPP Registrar Considerations

This document requires no interaction with the XMPP Registrar [12].
